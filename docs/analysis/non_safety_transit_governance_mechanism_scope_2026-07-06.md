# Transit Governance-Fit and Mechanism Scope — 2026-07-06

**Type:** dated governance-fit assessment and mechanism-scoping memo. Evaluates whether `transit` should become a deeper non-safety comparison group, per the open scoping decision flagged in `wage_mechanism_project_checkpoint_2026-07-05.md` §11 and `non_safety_comparison_roadmap_2026-07-04.md` §4.

## 1. Purpose and scope

This memo evaluates whether transit is a suitable non-safety comparison group for this project's city×time-fixed, occupation-varies design. It focuses on this project's nine current cities (per `data/city_coverage.csv` and `data/contracts.csv`), Massachusetts institutional context, and bounded national context. **This is governance-fit assessment and mechanism discovery, not corpus-building or a causal claim.** It does not ingest any document, does not download or store any corpus file, does not add any row to `data/contracts.csv` or `data/city_coverage.csv`, and does not run GABRIEL, model scoring, or any OEWS/BLS build. Every source below was found via bounded public web search and has not been verified for authenticity, completeness, or licensing status — none of that verification is performed or authorized by this memo.

## 2. Current project cities and corpus coverage

**No `transit` occupation-class rows exist.** A direct query of `data/contracts.csv` (32 rows, 7 distinct occupation classes present: `police`, `fire`, `teacher`, `sanitation` [zero], `clerical_admin`, `public_works`, `library`, `other`) confirms zero rows with `occupation_class = transit`. `data/city_coverage.csv` (32 rows) likewise contains zero `transit` rows.

**No transportation-adjacent rows exist either.** A keyword search of every field in `data/contracts.csv` for `transit`, `bus`, `transport`, `parking`, `traffic`, `operator`, `driver`, `mbta`, `rta`, `paratransit`, `shuttle`, and `school bus` returned only false-positive substring matches (the string `rta` matching inside words like "certain"); no row's `bargaining_unit_name`, `total_comp_note`, or any other field contains genuine transit-, bus-, transport-, traffic-, or parking-related content. This project's corpus currently has **no window at all** into transit, traffic, or parking work for any of its nine cities — a materially different starting point than every other non-safety group developed so far (teachers, DPW, clerical/admin, and library each had at least one already-collected row; even sanitation's scoping work could point to five already-collected `public_works` rows to re-read for bundled content).

**Implications for the current matched-city CBA design:** because this project's causal corpus is built entirely from municipal CBAs, and (per Section 3 below) none of this project's nine cities directly employs its own transit workforce, there is no existing or readily foreseeable municipal-CBA path to a transit row for any of these nine cities under the current design. This is a sharper version of the sanitation finding (where at least a minority of cities showed a plausible municipal-staffing path) — for transit, the governance structure itself, not merely a data-availability question, is the first-order obstacle. See Sections 8 and 11.

## 4. Why transit matters for the project

Transit is worth testing specifically because it is the first occupation group in this project's development sequence (teachers, DPW, clerical/admin, library, sanitation) whose primary obstacle is **governance structure**, not source availability or hazard/credentialing ambiguity. Compared with the prior groups:

- **Versus police/fire:** transit shares operational essentiality (a bus that doesn't run is a visible, immediate service failure, much like an unanswered 911 call) and, for MBTA workers specifically, a genuine compulsory-arbitration backstop (see §8) — making it, on paper, one of the closest institutional analogs to police/fire's JLMC backstop that this project has yet examined. But that backstop sits inside a completely different statute (M.G.L. c. 161A) governing a state authority's own employees, not a municipal bargaining unit this project's design is built to observe.
- **Versus sanitation:** sanitation's obstacle (per `sanitation_city_service_structure_scan_2026-07-06.md`) is that a majority of this project's cities contract collection to a private hauler, a city-level policy choice that varies city by city. Transit's obstacle is structurally deeper: Massachusetts state law itself (M.G.L. c. 161A for the MBTA; M.G.L. c. 161B for RTAs) assigns transit provision to state or regional authorities as a matter of statutory design, not a city-by-city contracting decision any of this project's nine cities could have made differently. No city in this scan "chose" to have no transit workforce of its own the way a city might choose to contract out trash collection.
- **Versus DPW:** DPW's contractor-substitution option (storm response, seasonal work) is a supplemental capacity choice layered on top of an otherwise-intact municipal DPW workforce. Transit's private-operator model (WRTA/CMTM) is not supplemental — for RTA-served cities, it appears to be the *entire* operating structure for the workforce, by statutory requirement.
- **Versus teachers, clerical/admin, and library:** those three groups all have at least one other feature (credentialing, low salience, professional degree requirements) this project has already mapped in depth; transit's CDL/commercial-driving requirement, split-shift scheduling, and public-facing-conflict/safety-risk profile would each be genuinely new mechanism territory for this project if a transit workforce were ever observable — but see §11 for why that observability question, not the mechanism content, is transit's actual bottleneck.

## 5. Transit occupation/classification map

Transit should not be treated as one occupation, both in general and specifically for what this scan found in this project's nine cities:

- **Bus operators** — the largest identified workforce category (MBTA's ATU Local 589 includes "nearly 6,000" bus, train, and maintenance employees; WRTA/CMTM's ATU Local 22 explicitly covers bus drivers). This is the category most directly analogous to a CDL-holding DPW equipment operator (H19/DP02-DP04), but under a wage-setting regime entirely outside this project's municipal-CBA design.
- **Rail/subway/trolley operators** — a distinct MBTA-specific category (Green Line, Red Line, Orange Line operators) not present at all in any RTA-served city in this scan; none of this project's nine cities has its own rail-operator workforce, since all rail service identified is MBTA-operated.
- **Paratransit drivers** — identified for GATRA (Franklin, Seekonk), MeVa (Georgetown), and MWRTA (Wayland) as Dial-A-Ride/demand-response service, a smaller, often senior/disability-eligible-rider-focused workforce than fixed-route bus operation; likely to have a different (possibly even more explicitly contracted-out) staffing model than fixed-route service, not independently confirmed this session.
- **Dispatchers/controllers** — not identified or investigated in depth for any transit entity in this scan; a genuinely distinct occupation from operators, mirroring this project's own already-flagged dispatcher classification question (checkpoint memo §9, §11) for police/fire-adjacent dispatch work.
- **Mechanics/maintenance workers** — MBTA's ATU Local 589 explicitly includes "repair and maintenance workers" alongside operators; not separately investigated for any RTA in this scan, but plausibly covered by the same operating-contractor arrangement as operators where one exists (e.g., CMTM at WRTA).
- **Station/customer-service workers** — relevant primarily to MBTA's subway/rail stations (Newton, Somerville, Arlington-adjacent Alewife); not identified as a distinct category for any RTA-served city, which have no rail stations of their own in this project's city set.
- **Supervisors** — not investigated separately in this scan for any entity.
- **Private-contractor operators** — the single most important classification distinction this scan surfaced: WRTA's ATU Local 22 workforce is, per M.G.L. c. 161B's operating-company requirement, technically employed by a private company (CMTM/First Transit), not by the public transit authority itself, even though the public authority sets policy, owns the service brand, and receives public subsidy. This distinction (public authority vs. private operating-company employer) is the transit analog to the causal-vs-discourse corpus distinction this project already enforces elsewhere: getting it wrong would misattribute a private-sector labor agreement to a public-sector wage-setting process.
- **School bus drivers** — not identified as present for any of this project's nine cities in this scan; if they exist, they are typically employed by a separate private vendor (e.g., a dedicated school-transportation contractor) under a school district's own procurement, not a general municipal transit function, and should be kept structurally distinct from public transit per the task's explicit instruction. This scan did not investigate school-transportation arrangements for any of the nine cities.
- **Traffic/parking/transportation-department employees** — confirmed present and structurally distinct from transit operations for at least Boston (the Boston Transportation Department, responsible for "traffic and parking," explicitly not a transit-operations function). This category is municipal (a genuine city department), but it is not transit work and should not be coded as such if ever encountered in a future corpus document; it is closer in kind to a DPW-adjacent municipal function than to bus/rail operations.

## 6. Upward wage-pressure mechanisms

- **Route/service coverage.** A missed bus trip or a canceled route is a dated, individually-experienced, publicly visible service failure, structurally similar to sanitation's missed-route mechanism (SN01/H32) and, more distantly, to police/fire's coverage-cannot-be-deferred logic. Not independently tested against any actual transit CBA this session, since none is in this project's corpus.
- **Staffing shortages and missed trips/canceled service.** The MBTA's own recent bus-operator wage increase (starting pay raised from $22.21 to $30/hour) was explicitly tied, per this session's search results, to a hiring-difficulty problem severe enough that operator applications subsequently surged 365% after the raise — a striking, if MBTA-specific and very recent (2024), demonstration of a direct wage-to-staffing-recovery relationship this project has not yet observed this cleanly for any other occupation group, including police/fire.
- **CDL/commercial driving.** Bus operation requires a commercial driver's license with passenger endorsement, structurally analogous to DPW's CDL-linked classification/stipend mechanism (H19) and sanitation's CDL-driver discussion (SN03) — not yet tested for whether any transit entity ties CDL possession to a distinct pay tier, since no transit CBA is in this project's corpus.
- **Safety-sensitive work.** Federal transit-worker drug-and-alcohol testing regulations (a general national requirement for safety-sensitive transit positions, not independently verified in detail this session) and public reporting of bus-operator assaults are both plausible textual signals; not independently confirmed for any of this project's nine cities' transit entities this session.
- **Public-facing conflict.** Bus operators interact directly and continuously with the public in a way DPW/sanitation workers typically do not, a distinct exposure profile from both police/fire's enforcement-context conflict and DPW's infrastructure-only work; not independently tested this session.
- **Split shifts/irregular schedules.** A commonly reported feature of bus-operator scheduling nationally (not independently verified for any of this project's nine cities' transit entities this session); plausible textual signal for any future transit CBA review.
- **Overtime.** Not independently tested this session for any transit entity; plausible given fixed-route service's continuous-coverage requirement, similar in logic to police/fire and DPW overtime/staffing-spiral mechanisms (XC03).
- **Mechanics/maintenance skills.** MBTA's ATU Local 589 explicitly includes maintenance workers; a CDL/skilled-trades-adjacent credentialing story, similar in kind to DPW's mechanic/equipment-operator classification tiers (DP09, DP10), not independently tested for any RTA this session.
- **Private-sector competition.** Not independently tested this session; plausible given the general trucking/commercial-driving labor market DPW's own CDL discussion (H19/DP02) already treats with explicit skepticism about "shortage" framing.
- **Public salience from service disruptions.** A canceled Green Line branch or a systemwide service suspension (the MBTA's own November 2025 Green Line suspension through Boston, Cambridge, and Somerville, identified in this session's search results as a live, recent example) is a visible, dated, attributable service failure with clear potential political salience — a genuinely testable transit-specific analog to police/fire's public-safety-salience hypothesis (H7/XC10), though this scan did not test whether such disruptions actually generate comparable political/budgetary pressure to a police/fire incident.
- **Union strength.** The 2024 MBTA settlement — the first time in 15 years the MBTA reached agreements with all 28 of its affiliated labor unions simultaneously, following a specific, unusually large wage increase for the largest union (ATU Local 589) — is a striking recent illustration of union bargaining leverage translating directly into a large wage outcome, though this is explicitly an MBTA-specific (state-authority), not municipal, bargaining event, and should not be read as evidence about any of this project's nine cities' own wage-setting processes.

## 7. Wage-restraint or alternate wage-translation mechanisms

- **Regional/state governance rather than city bargaining.** This is transit's dominant, distinguishing restraint mechanism relative to every other occupation group this project has developed: for all nine of this project's cities, the entity that actually negotiates transit-worker wages (the MBTA, or a regional authority's private operating contractor) is not the city government whose CBAs make up this project's causal corpus at all. This is not a "restraint" in the sense DPW's contractor substitution or clerical/admin's classification system restrain wage growth *within* an otherwise-observable municipal bargaining process — it removes the wage-setting process from this project's observable frame entirely, the same structural point already made for sanitation's private-hauler cities (§4-5 of `sanitation_city_service_structure_scan_2026-07-06.md`), but here applying to all nine cities without exception, not a subset.
- **Contractor/private-vendor operation.** WRTA's CMTM/First Transit arrangement is the clearest documented instance: the public transit authority sets service policy and receives public subsidy, but a private company is the actual employer of record for the unionized workforce. Under M.G.L. c. 161B, this structure is not a discretionary choice available to some RTAs and not others — it is described in this session's search results as a requirement ("must hire private operating companies") for RTAs generally, meaning it plausibly extends to GATRA, MeVa, and MWRTA as well, though this was independently confirmed only for WRTA this session.
- **Farebox/subsidy/budget constraints.** WRTA's fare-free policy (in place since March 2020, extended through at least June 2026 per this session's search results) shifts the transit authority's entire revenue structure toward public subsidy — a genuinely distinct fiscal-capacity story from any municipal department's tax-levy-based budget constraint (H9/XC11), not independently tested for its wage-setting implications this session.
- **Route cuts/service reductions.** Not independently tested this session for any of this project's nine transit entities; plausible structural analog to DPW's service-deferral mechanism (DP08/H21) and sanitation's route-coverage-pressure discussion (SN01/H32), but for a state/regional authority's own budget process, not a municipal one.
- **Schedule redesign.** The MBTA's own "Bus Network Redesign" project, identified in this session's search results as actively restructuring routes serving Arlington and elsewhere, is a concrete, current example of large-scale schedule/route restructuring; not independently tested for wage implications this session.
- **Paratransit contracting.** GATRA's, MeVa's, and MWRTA's Dial-A-Ride/paratransit services are plausibly contracted through the same or a similar private-operator model as fixed-route service; not independently confirmed for any of the three this session.
- **Classification/internal equity.** Not independently tested this session for any transit entity; a plausible mechanism given this project's confirmed finding of classification-based wage restraint in DPW (DP09/H22) and clerical/admin (CA01/H23).
- **Capital vs. operating budget constraints.** A genuine, transit-specific fiscal distinction (vehicle/infrastructure capital costs versus day-to-day operating costs, often funded through different mechanisms — federal/state capital grants versus farebox/local-assessment operating revenue) not independently tested this session but structurally analogous to DPW's capital-versus-operating-budget service-deferral discussion (DP08/H21).
- **Governance distance from municipal police/fire bargaining.** The clearest, most general restraint-adjacent finding of this entire scan: transit wage-setting for every one of this project's nine cities occurs through a process (MBTA's own c. 161A arbitration; an RTA's private operating contractor's own labor-relations process) that is institutionally further removed from any of this project's municipal police/fire bargaining processes than any other occupation group this project has examined — DPW, clerical/admin, teachers, and library are all at least nominally under the same general municipal Chapter 150E §9 process as this project's non-safety comparison logic assumes; transit, for every city in this scan, is not.
- **Service changes as buffering.** Not independently tested this session; plausible general category encompassing several of the above (route cuts, schedule redesign, paratransit contracting) as alternatives to wage growth.

## 8. Massachusetts institutional context

**MBTA governs transit for Arlington, Boston, Newton, and Somerville, and it is a state authority, not a municipal employer.** The MBTA is organized under M.G.L. c. 161A as a division of the Massachusetts Department of Transportation (MassDOT). Its unionized workforce (ATU Local 589 for bus/rail operators and maintenance; other affiliated unions including at least one machinists' local, per this session's case-law search results) bargains directly with the MBTA itself, not with any city government. None of this project's four MBTA-served cities employs its own transit workforce in any form identified this session.

**Massachusetts regional transit authorities (GATRA for Franklin and Seekonk; MeVa/MVRTA for Georgetown; MWRTA for Wayland; WRTA for Worcester) are themselves public entities, but M.G.L. c. 161B requires them to contract actual service operation to private operating companies.** This session directly confirmed this structure for WRTA specifically: First Transit, Inc., incorporated in Massachusetts as Central Mass Transit Management (CMTM), is WRTA's operating contractor and the direct employer of WRTA's unionized (ATU Local 22) bus operators. This session's search results describe the private-operator requirement as a general feature of the RTA model ("RTAs, which are locally controlled, manage their own operations but must hire private operating companies to provide their services in accordance with Chapter 161B"), which would extend the same structure to GATRA, MeVa, and MWRTA, though this was not independently confirmed for those three specific authorities this session.

**A genuine, previously undiscovered institutional finding: the MBTA has its own compulsory interest-arbitration statute, separate from both JLMC and Chapter 150E.** M.G.L. c. 161A §§19C-19G (added in 1978) prescribes compulsory interest-arbitration procedures for impasses between the MBTA and its unions, and c. 161A §19 (per the 1980 management-rights amendment, Acts of 1980 c. 581 §8) separately preserves a contractual grievance-arbitration route. This means MBTA transit workers are **not** in the same institutional position as this project's other non-safety groups (teachers, DPW, clerical/admin, library — all confirmed, four separate times, to follow the ordinary Chapter 150E §9 route with no compulsory-arbitration backstop). MBTA transit workers instead have a compulsory-arbitration backstop of their own — just one that sits under an entirely different statute (c. 161A), governs a state authority's own employees rather than any municipality's, and has never been examined by this project before this session. **This is a genuinely new, three-way institutional finding for this project** (JLMC for police/fire; Chapter 150E §9 for teachers/DPW/clerical-admin/library/sanitation-by-extension; c. 161A for MBTA transit workers specifically) rather than a simple confirmation of the already-established police/fire-only JLMC pattern.

**No transit-specific JLMC-style backstop applicable to any city government was identified in this bounded review.** This finding is not in tension with the c. 161A finding above — c. 161A governs the MBTA as an employer in its own right, not any of this project's nine city governments, none of which employs a transit workforce directly. If a future session ever needed to code a "does this occupation have a compulsory-arbitration backstop" attribute for transit, the honest answer is genuinely bifurcated by governance type: MBTA-employed transit workers (Arlington, Boston, Newton, Somerville's service area) have one, via c. 161A; RTA-operating-contractor-employed transit workers (Franklin, Georgetown, Seekonk, Wayland, Worcester's service area) are private-sector employees whose wage-setting process this session did not investigate in enough depth to characterize (plausibly NLRA-governed private-sector bargaining, not any Massachusetts public-sector statute at all — see the RTA-employee NLRA-status ambiguity flagged in this session's search results, which found no source directly confirming RTA-operating-company employees' precise labor-law classification).

**Ordinary public-sector labor/impasse context, restated briefly per the task's own instruction not to overdo legal analysis:** to the extent any transit-adjacent worker in this project's nine cities were ever a genuine city or town employee (e.g., a hypothetical municipal Transportation Department traffic/parking staffer, as opposed to a transit operator), that worker would be expected to follow the same ordinary Chapter 150E §9 process already independently verified four times for this project's other non-safety groups — this was not separately re-verified for any specific traffic/parking department this session, since no such document is in this project's corpus and none was identified as an imminent acquisition target.

## 9. National context

**Staffing shortages.** The MBTA's own 2024 experience (bus-operator starting pay raised from $22.21 to $30/hour, followed by a reported 365% surge in applications) is consistent with widely reported post-pandemic U.S. transit-operator staffing shortages generally, though this session did not independently verify a national data source (e.g., APTA or a comparable national transit-workforce survey) beyond the MBTA-specific reporting already found; this remains `partially searched` at the national level.

**Operator assaults/safety.** Not independently investigated at the national level this session; a commonly cited national transit-workforce concern in general reporting, not verified with a specific source this session.

**CDL/commercial driving.** Bus and paratransit operation nationally requires a commercial driver's license with passenger endorsement, structurally analogous to (and potentially competing in the same labor pool as) the CDL-linked mechanisms already documented for DPW (H19) and scoped for sanitation (SN03); not independently investigated with a national-level source this session beyond what those two prior memos already establish generally about CDL labor-market claims.

**Service cuts/missed trips.** The MBTA's own recent (November 2025) systemwide Green Line suspension through Boston, Cambridge, and Somerville, identified in this session's search results, is a concrete, current, Massachusetts-specific example of a major service disruption; not independently benchmarked against a national database of transit service disruptions this session.

**Maintenance skills.** Not independently investigated at the national level this session beyond noting that MBTA's ATU Local 589 explicitly includes maintenance workers as part of its represented workforce.

**Public subsidy constraints.** WRTA's fare-free policy (in place since March 2020) is a Massachusetts-specific example of a transit authority's revenue structure shifting toward public subsidy; not independently benchmarked against a national fare-free-transit trend or database this session, though fare-free transit has been reported as a growing national policy trend in general secondary reporting outside this session's specific searches.

**Operator scheduling.** Not independently investigated at the national level this session.

**Contracting.** M.G.L. c. 161B's private-operating-company requirement for Massachusetts RTAs is consistent with a broader, commonly reported national pattern of transit agencies contracting fixed-route or paratransit operations to private management firms (First Transit itself, per this session's search results, operates transit systems in multiple U.S. states, not just Massachusetts); not independently benchmarked against a systematic national count of publicly-operated versus privately-contracted transit systems this session.

## 11. Governance-fit assessment

**Is transit a good comparison group for the current city-level matched CBA design? No, not as currently defined.** Every one of this project's nine cities' transit service is governed and staffed either by the MBTA (a state authority, under M.G.L. c. 161A, with its own distinct compulsory-arbitration statute) or by a regional transit authority whose actual operating workforce, per M.G.L. c. 161B and this session's WRTA-specific confirmation, is employed by a private operating company, not the public authority itself and certainly not any city government. This project's causal corpus is built entirely from municipal CBAs (one row = one bargaining unit's contract, for one negotiation cycle, in one city, per `CLAUDE.md`'s unit-of-observation rule); transit, for every city in this scan, does not have a municipal bargaining unit of this kind to observe at all.

**Is transit better treated as a separate institutional comparison rather than a direct matched-city occupation? Yes, if pursued at all.** The most defensible way to use transit's genuinely interesting institutional findings (the MBTA's own c. 161A compulsory-arbitration statute; the RTA private-operator model) would be as a **separate institutional case study** — parallel to, but not merged into, this project's city×time-fixed occupation comparison — documenting how Massachusetts handles wage-setting for another operationally essential, public-facing occupation entirely outside the municipal bargaining framework this project's main design depends on. Forcing a transit row into `data/contracts.csv` under the current schema (one row = one city's bargaining unit) would misrepresent either an MBTA-wide state agreement or a private operating-contractor's private-sector CBA as if it were a "city's" contract, which it is not, for any of this project's nine cities.

**Should transit be included now, deferred, or included only if city-operated cases are found? Deferred, with one narrow exception to watch for.** This scan found zero `city_operated_transit` cases among this project's nine cities — every city fell into `state_transit_authority`, `regional_transit_authority`, or `mixed_public_private` (with the private-operator layer confirmed specifically for Worcester/WRTA). If a future scan of additional Massachusetts cities (beyond this project's current nine) ever identified a city that directly operates its own municipal transit/shuttle service with its own municipal bargaining unit, that specific city could in principle be added under the existing schema without any special-case design change — but no such case exists among this project's current nine cities, and this scan did not search beyond them (per the task's explicit scope boundary).

**Which current project cities, if any, are promising transit-source targets? None, for the causal (municipal-CBA) corpus, under the current design.** If a future session wanted to develop transit as the separate institutional case study recommended above (not a matched-city occupation), the most information-rich targets identified this session would be: (1) the MBTA's own ATU Local 589 CBA and its c. 161A arbitration case history (the richest, most directly quotable institutional material found in this scan, and the source of the genuinely new three-way arbitration-regime finding in §8); (2) WRTA's CMTM/ATU Local 22 arrangement (the clearest documented instance of the RTA private-operator model, and the only RTA in this scan with an operating-contractor identity independently confirmed). Both would require their own explicit authorization before any acquisition, consistent with this project's standing ingestion-layer rules, and neither would be classified as a `data/contracts.csv` row without a prior schema/design decision from the user/PI, since neither is a municipal bargaining unit for any of this project's nine cities.

## 12. New or refined hypotheses

The following hypotheses are proposed for future tracking (see Task H for which, if any, were added to the hypothesis matrix as new rows):

- **transit_route_service_coverage_pressure** — a missed-trip/canceled-service analog to sanitation's route-coverage hypothesis (SN01/H32) and, more distantly, police/fire's coverage-cannot-be-deferred logic.
- **transit_operator_shortage_and_missed_service** — the MBTA's 2024 wage-raise-then-hiring-surge pattern, the sharpest quantified wage-to-staffing link found in this scan, though describing a state authority's own workforce, not a municipal one.
- **transit_safety_sensitive_public_facing_work** — operator-assault/public-conflict exposure, not independently evidenced for any of this project's cities this session.
- **transit_split_shift_schedule_burden** — a nationally-reported scheduling-burden pattern, not independently confirmed for any of this project's cities' transit entities this session.
- **transit_contracting_governance_distance** — this scan's single most consequential and best-evidenced finding: transit wage-setting sits outside this project's municipal-CBA design for every one of its nine cities, via either state-authority (MBTA) or private-operating-contractor (RTA) governance, a structurally deeper version of the contractor-substitution/governance-distance logic already documented for DPW (XC06) and sanitation (H33).
- **transit_service_cut_buffering** — route cuts/schedule redesign as a wage-alternative buffering channel, not independently tested this session.
- **transit_regional_authority_budget_constraint** — farebox/subsidy-driven fiscal structure (e.g., WRTA's fare-free policy) as a distinct fiscal-capacity story from municipal tax-levy constraints.
- **transit_mechanic_maintenance_skill_pressure** — a CDL/skilled-trades-adjacent credentialing story for transit maintenance workers, structurally close to DPW's mechanic-classification mechanisms (DP09, DP10).
- **non_safety_wage_restraint (H17)** — transit adds a genuinely new wrinkle, sharper than sanitation's: for sanitation, the workforce exists but may sit outside the municipal-CBA frame depending on a city's contracting choice; for transit, the workforce structurally sits outside that frame for every single one of this project's nine cities, by state-law design rather than city-level choice.

## 14. Recommended next step

**Do not pursue targeted transit source acquisition as this project's next comparison-group development step.** Transit's central finding — governance distance from any municipal bargaining unit, for all nine of this project's current cities, without exception — is a scoping conclusion, not merely an unresolved data-availability question the way sanitation's was. Unlike sanitation (where Seekonk offered an immediate, zero-cost path forward), this scan identified no comparably low-cost within-design path for transit.

**Recommend deferring transit indefinitely as governance-mismatched for this project's current city-level matched-CBA design**, while flagging the MBTA's c. 161A arbitration statute and the RTA private-operator model as genuinely interesting, separately-documentable institutional facts that could inform a future, explicitly non-matched-city institutional case study, should the user/PI want one — but that is a distinct decision from continuing this project's occupation-by-occupation comparison-group sequence.

**Recommend moving to nurses/health scoping or custodial/facilities scoping next**, per the still-open items in `wage_mechanism_project_checkpoint_2026-07-05.md` §11, both of which remain pending a user/PI scoping decision independent of this transit assessment.
