# Report Review Checklist — Deeper Look Into Safety & Non-safety Wage Mechanisms

**Type:** planning/review document. Use this alongside `report_draft_safety_non_safety_wage_mechanisms_2026-07-06.md` before any formatting or artifact-generation run.

## 1. Claims that need user/PI review

- The Executive Summary's framing of the "central finding" (pressure-to-wage conversion, not difficulty of work) is the report's single most load-bearing claim. Confirm this is the framing the PI wants foregrounded, rather than, say, an institutional-history framing or a pure descriptive-inventory framing.
- The group-retention labels (central / strong comparison / public-safety-adjacent / secondary / source-design case / deferred) were supplied as a fixed frame for this run. Confirm the PI is comfortable with dispatchers and nurse_health's placement specifically — both are recent upgrades from a prior "defer" status, and the PI may want to weigh in on how much prominence a single-city finding (nurse_health) or a civilian-but-public-safety-adjacent group (dispatchers) should receive in a report this early in the project's life.
- The claim that custodial/facilities' evidentiary work is "largely complete" (Evidence Map, Group-by-Group section) is accurate for the two fully-documented cities but should be reviewed against whether the PI wants any additional cities before treating this group as settled.
- The Massachusetts-vs-national nuance section states that Iowa is a confirmed exception to the "non-safety lacks compulsory arbitration" pattern. Confirm the PI wants this counterexample foregrounded in the main text rather than moved to a footnote-level caveat, since it complicates the report's central institutional claim.

## 2. Claims that need source strengthening before final version

- **Dispatcher wage-tier comparison (H43 / dispatch civilian-status undertranslation):** the report states this comparison has not been completed. If the PI wants this claim strengthened before a later, more polished version, the specific next step (a same-city dispatcher-vs-police/fire wage-table comparison, most feasible in the city with a full dispatcher wage-grade table) is already identified in Section 7.
- **Nurse_health generalizability:** the report's single-city finding should not be extended to any general statement about municipal nursing without additional cities. Flag any future draft that implicitly broadens this claim.
- **Sanitation's "possible but unconfirmed" signals:** the report accurately hedges the transfer-station and CDL-training signals as suggestive, not confirmed. Any future version should not upgrade these without a specific new confirmation step.
- **Transit's wage-to-staffing finding (a 2024 wage increase followed by an application surge):** this is one data point from one state authority. The report already flags this explicitly; watch that a later editorial pass does not smooth this into a more general statement about wage elasticity.
- **National generalizability claims throughout Section 6:** every national claim in the draft is sourced to the project's own citation-audit work. Any new national claim added in a later draft should go through the same citation-confidence discipline (primary source vs. secondary-source-only vs. legally unsettled) before inclusion.

## 3. Formatting decisions still open

- Whether the Executive Summary should be 3, 4, or 5 paragraphs in the final version (the current draft uses 4; the task allowed 3-5).
- Whether Main Takeaways should be numbered 1-8 as in the current draft, or trimmed to a tighter 5-6 for a shorter reading experience.
- Whether the Evidence Map table's column widths will need adjustment once the Georgia/11pt style guide is applied — the "Strongest Current Evidence" and "Key Caveat" columns are the longest and may need to wrap onto multiple lines at letter-page width.
- Whether section numbers (used in this draft for internal cross-referencing only) should appear in the final formatted version, or be replaced with unnumbered heading style consistent with the accent-color/heading-hierarchy guide.
- Whether the appendix should use the same table styling as the main text or a more compact, smaller-type convention, given its higher table density.

## 4. Tables that may be too large and should perhaps move to appendix

- The **Mechanism-by-Mechanism Synthesis** section's implicit table structure (13 mechanisms × 5 sub-fields each) is long in the main text as written prose; consider whether a condensed summary table (mechanism × strongest-groups × evidence-status, similar to Appendix B) should be added near the top of that section as a navigational aid, with the full prose treatment following.
- The **Source Needs / Next Steps** section currently has three separate tables (must-have, useful, optional/deferred). Consider whether the optional/deferred table could move entirely to the appendix, leaving only must-have and useful in the main text, to keep the section shorter.
- **Appendix D (hypothesis disposition summary)** is currently prose-only, deliberately not reproducing the full 43-row disposition table. Confirm this is the right level of detail for an appendix — a condensed table (grouped by recommendation category rather than by individual hypothesis ID) is a plausible middle ground if the PI wants more structure than prose but less than the full CSV.

## 5. Sections that may need tightening

- The **Group-by-Group Findings** section is the longest in the draft (11 groups × 7 sub-fields). If the PI wants a shorter report, this is the first section to compress — candidates for trimming include the "National-generalizability caveat" line for groups where no national comparison was attempted at all (library, custodial/facilities), which could be folded into a single sentence rather than a full field.
- The **Massachusetts and National Nuance** section's second paragraph (national context caveats) partially duplicates caveats already stated in the Group-by-Group entries for police, fire, and teachers. Consider whether to cross-reference rather than restate.
- Several Group-by-Group entries (dispatchers, nurse_health, sanitation) carry unusually long "Current evidence basis" fields relative to other groups, reflecting how recently and thoroughly those groups were reviewed. Consider whether to normalize length across all 11 groups for visual consistency, even where the underlying evidence genuinely differs in depth.

## 6. Suggested questions for PI/user

1. Does the group-retention frame (central / strong comparison / public-safety-adjacent / secondary / source-design case / deferred) match how you want to present this work, or would a simpler two-tier frame (e.g., "developed" vs. "in progress") read better to an outside audience?
2. Should the report name specific cities in the main text (as the current draft does implicitly through phrasing like "one city's contract"), or should city names be reserved for the appendix/footnotes to keep the main text more general?
3. Is the current must-have source-needs list (three items) the right scope for a next research step, or would you rather see a broader acquisition plan drafted alongside this report?
4. Do you want a version of this report that includes a short "what this report does not claim" callout box near the front, given the emphasis on avoiding causal language?
5. Should transit receive more or less space than the current draft gives it, given it is the group with the least direct evidence but arguably the most analytically interesting governance finding?
