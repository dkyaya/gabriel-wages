# PI Note: Descriptive Police/Fire Wage-Trend Baseline

**Date:** 2026-07-03  
**Scope:** descriptive planning note only

The next empirical step should be a descriptive wage-trend baseline using public federal data. The purpose is not to identify causes yet. It is to establish what broad occupation-level wage patterns look like for police, fire, and selected municipal comparison occupations before the project invests in more mechanism-specific text measurement or more intensive contract-based wage construction.

The cleanest federal starting point is the BLS Occupational Employment and Wage Statistics program. OEWS provides annual employment and wage estimates for occupations at the national, state, and metropolitan/nonmetropolitan-area levels. That makes it well suited for a first descriptive panel covering the United States, Massachusetts, and the Boston-Cambridge-Newton metro. In parallel, Census ASPEP can provide government-function payroll context for police protection, fire protection, education, and other public-sector functions. That is useful for showing how public-sector staffing/payroll context moves around the same period, even though it is not a direct wage-setting dataset.

This baseline can answer a limited but important set of questions. It can show whether broad police and firefighter occupational wages move differently from selected comparison occupations over time at national, Massachusetts, and Boston-metro scales. It can also show whether Massachusetts appears unusual or ordinary relative to the national pattern, and whether broad government-function payroll context moves in a way that is consistent with increased public-safety wage pressure. That kind of baseline complements the mechanism memo by giving the project a descriptive wage frame before asking which mechanisms matter most.

Just as important, this baseline cannot answer several questions that the project ultimately cares about. OEWS does not observe bargaining units. It does not recover negotiated contract schedules. It does not tell us what happened to a specific municipal unit such as Boston police patrol, Franklin fire, or a matched clerical unit in the same bargaining cycle. ASPEP is even further from that target: it is function-based employment and March payroll data, not bargaining-unit wage schedules. So the baseline is useful precisely because it is modest. It provides a consistent descriptive frame without claiming to measure unit-level contract wages.

That is why unit-level contract wages still require separate CBA or payroll work. To compare police/fire units against non-safety units within the same city and cycle, the project will eventually need negotiated wage tables from CBAs or clearly public salary/payroll files that can be aligned to actual bargaining units. Federal occupation tables are not substitutes for that. They are a first descriptive layer that helps us see the broad terrain before doing the harder within-city unit-level wage construction.

The practical recommendation is therefore:

1. build a first OEWS panel for police, firefighters, teachers, clerical/admin proxies, public-works proxies, sanitation, and transit where feasible;
2. add ASPEP function-based context for police protection, fire protection, education, and other relevant functions;
3. keep the interpretation descriptive;
4. treat unit-level contract wage trends as a separate second-stage task.

This sequence fits the current project logic well. The mechanism memo widened the conceptual frame beyond a single comparability story. A descriptive wage-trend baseline is the right next step because it adds empirical structure without prematurely collapsing that wider mechanism space into a causal claim.
